# Plan: Corrección de Mejores Prácticas + Demo Wallet con Transacciones Reales

## Hallazgos del Code Review

Se encontraron **22 issues**: 2 críticos, 5 altos, 5 medios, 10 bajos.

### Críticos (arreglar ya)
1. **CSS duplicado** — `styles.css` tiene 2 copias de muchos bloques (demo-section, result-card, policy-bar, etc.) con valores divergentes. ~400 líneas de código muerto.
2. **`let eventCounter = 0`** — estado mutable a nivel módulo. En StrictMode incrementa de a 2. Anti-patrón React.

### Altos (arreglar ya)
3. Sin Error Boundary — si un componente crashea, pantalla en blanco
4. Sin cleanup de timeouts en `runAnalysis` — memory leak
5. Feed items del security log no son accesibles por teclado
6. Inputs sin `aria-label`
7. Links externos sin `rel="noreferrer"`

### Medios
8. Regex `\\n` no matchea newlines reales
9. `document.getElementById` en vez de `useRef`
10. Race condition entre scroll timeout y `setBusy(false)`
11. Inline styles mezclados con clases CSS
12. App component muy grande (~435 líneas)

---

## Plan de Implementación

### Fase 1: Limpieza CSS y fixes críticos
- **Eliminar CSS duplicado** — borrar bloques repetidos, dejar 1 copia canónica
- **`eventCounter` → `useRef`** dentro del componente App
- **Agregar ErrorBoundary** component
- **Agregar cleanup** con AbortController en `runAnalysis`
- **A11y**: `aria-label` en inputs, `role="button"` + `tabIndex` + `onKeyDown` en feed items
- **`rel="noopener noreferrer"`** en todos los links externos
- **Regex fix**: `\\n` → `\n` (o remover si no es necesario)
- **`useRef`** para scroll en vez de `getElementById`
- **Mover inline styles** a clases CSS

### Fase 2: Demo Wallet — Transacciones Reales
- **Agregar `VITE_DEMO_PRIVATE_KEY`** a `frontend/.env.local`
- **Crear `src/demo-wallet.ts`** — hook `useDemoWallet()` con:
  - `privateKeyToAccount` para crear cuenta desde private key
  - `createWalletClient` + `createPublicClient` con http transport
  - Funciones: `setSecurityProfile`, `analyze`, `fulfillManual`, `getDecision`
  - Sin wagmi, sin MetaMask — puro viem
- **Agregar toggle "Live Mode"** en la UI:
  - 🔬 Demo Mode (default): simulación client-side como ahora
  - ⛓️ Live Mode: transacciones reales on-chain con la demo wallet
- **En Live Mode**:
  - Mostrar balance STT de la demo account
  - Tx hashes reales + links al explorer
  - Las 4 etapas del pipeline son transacciones reales
  - Manejar errores: insufficient funds, tx rejected, timeout
  - Badge "⛓️ Live" en resultados

### Fase 3: Split de Componentes
- Extraer `PolicyEditor`, `DemoScenarios`, `AnalyzeSection`, `SecurityLog`, `StatsFooter`
- App queda como orquestador (~150 líneas)

La app tendrá **dos modos**:

| | 🔬 Demo Mode | ⛓️ Live Mode |
